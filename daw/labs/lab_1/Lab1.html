Laboratorio 1

URL

(Uniform Resource Locator). Dirección única de un recurso dentro del internet.

MIME

(Multi-Purpose Internet Mail Extensions). Extensión del protocolo e-mail que le permite a los usuarios intercambiar diferentes tipos de archivos y datos en el internet.

Métodos y códigos de errores

HTTP

Métodos
  • GET
    • Adquirir recurso de servidor web.
  • HEAD
    • Sólo solicita los metadatos del recurso.
  • POST
    • Enviar información al servidor como subelemento de un recurso ya existente.
  • OPTIONS
    • Averiguar qué métodos HTTP soporta el servidor.
  • PUT
    • Cargar nuevo recurso al servidor.
  • DELETE
    • Borrar recurso.
  • TRACE
    • Monitorear mensajes entre cliente y servidor.
  • CONNECT
    • Solicitar conexión de tipo túnel TCP/IP.
Errores
  • 4XX
    • Errores de cliente. Suceden cuando la página no existe o el acceso está bloqueado.
  • 5XX
    • Errores del servidor. Errores en el código de la página o dentro del propio servidor.

XML

Especificación para diseñar lenguajes de marcado, que permite definir etiquetas personalizadas para descripción y organización de datos.

XHTML

(Extensible Hyper Text Markup Language). Casi identico a HTML, pero más estricto. Es HTML difinido por una aplicación XML.

HTML5

Contiene nuevos elementos semánticos como "header", "footer"; nuevos atributos de elementos como "number", "date", "time"; nuevos elementos gráficos: "svg", "canvas"; elementos multimedia: "audio" y "video".

JS

JavaScript es el lenguaje utilizado para controlar el comportamiento de los documentos HTML. Puede modificar atributos, estilos, elementos, etc.
En JS se utiliza la palabra clave "function" para definir un bloque de código diseñado para ejecutar una tarea en particular. Las funciones pueden incluir varios parámetros.
function nombre(p1, p2, p3, ...){ código }.

PHP

Lenguaje para servidores y para crear páginas web dinámicas e interactivas. PHP cuenta con varias funciones para arreglos, operaciones matemáticas, Strings, HTTP, XML, MySQLi, etc. También cuenta con funciones para definir cómo debe reaccionar la página o el servidor al encontrarse un error.

JSP

(Java Server Pages). Utilizado para crear aplicaciones web que vayan a ser ejecutadas en varios servidores,de múltiples plataformas. Las páginas JSP están compuestas de HTML,XML y scripts en Java.

Ajax

(Asynchronous JavaScript And XML). Es una combinación de objetos XMLHttpRequest, JavaScript y HTML DOM. Permite a las páginas ser actualizadas de manera asíncrona, ya que intercambia información con el servidor detrás de cámaras. Hace posible actualizar partes de una página sin tener que refrescarla.

¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet es una red de redes, conecta millones de equipos entre sí, los cuales se pueden comunicar por medio de protocolos.
La World Wide Web es una manera de acceder a la información dentro del internet. Es un modelo de intercambio de información costruido sobre el internet. Utiliza el protocolo HTML. Es una porción del internet.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Petición de datos de un recurso específico.
HEAD: Igual que GET, pero sólo regresa headers HTTP, no regresa el body del documento.
POST: Enviar data para ser procesada por un recurso específico.
PUT: Sube una representación de URI específico.
PATCH: Aplica modificaiones parciales a un recurso.
DELETE: Elimina el recurso especificado.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML como el de la encuesta que constestaste al principio de la actividad? ¿Por qué?

POST, para que los datos enviados vayan de forma oculta.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

GET.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No, el código 200 representa una respuesta estándar para peticiones correctas.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Generalmente no, este error significa que la página no fué encontrada, algo que puede suceder por escribir mal la dirección por parte del usuario.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, indica un error interno. Existe código erroneo dentro del sitio.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Un atributo desaprobado es el cual ya no se recomienda su uso en los documentos HTML. Se siguen soportando para proveer compatibilidad con las versiones antiguas de HTML; sin embargo, son reemplazados por otro más actualizado. Los tags como "dir", "font", "u", son desaprobados en HTML5.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Los documentos HTML4 contienen tanto la estructura como los estilos aplicados al mismo. En HTML5 los documentos sólo contienen la estructura de la página, los estilos se configuran desde otro archivo dedicado para eso.

¿Qué componentes de estructura y estilo tiene una tabla?

"table" - contenedor principal.
"tr" - filas contenedoras de las celdas.
"td" - representa las celdas.
"th" - celdas de encabezado.

¿Cuáles son los principales controles de una forma HTML5?

"form", input", "email", "url", "date", "time", "range", "color", list", "placeholder", "autofocus", "required", "step", "min", "max", "autocomplete".

Ciclo de vida y desarrollo de los sistemas de información

Etapas del ciclo

  • Planificación
    • Tareas y actividades previas al comienzo del proyecto. Entre estas están: organización de tiempos, distribución de trabajo, herramientas a utilizar, estudio de factibilidad, etc.
  • Análisis
    • Aclarar cuáles son las funciones exactas que debe tener el sistema. Se descubre qué se necesita para poder plasmarlo en requerimientos del sistema.
  • Análisis de riesgos
    • Evaluación de posibles riesgos a la hora de estar desarrollando el sistema y durante su implementación.
  • Delimitación del ámbito del proyecto
    • Determinar lo que va a abarcar el proyecto, así como los aspectos que no forman parte de este.
  • Diseño
    • Se explora cómo el sistema cumplirá con los requerimientos identificados. Es un proceso iterativo, conforme vaya avanzando el desarrollo del sistema, el diseño del mismo irá cambiando naturalmente. Se busca encontrar la mejor alternativa para la implementación de los requisitos.
  • Implementación
    • Se instalan las herramientas a utilizar y se comienza con el desarrollo. Durante esta etapa es necesario documentar sobre la manera en que se está codificando, explicar cómo y porqué se realizó de esa manera.
  • Pruebas
    • Todo requisito implementado debe ser probado para poder detectar cualquier error y corregirlo antes del despliegue del sistema. Las etapas de implementación y de pruebas van intercalándose hasta que se obtenga cada función implementada correctamente.
  • Despliegue
    • Instalación del sistema y construcción de todos los archivos necesarios para su uso. Es recomendable desplegarla primero en sólo una sección de una empresa para hacer verificaciones de uso correcto, antes de que sea utilizada de manera general.
  • Mantenimiento
    • Eliminar defectos que surjan durante la vida útil del sistema. Adaptarlo, si se requiere, cuando se tiene un sistema que opera sobre un OS y este es actualizado. Optimizar y añadir funcionalidades con actualizaciones del sistema.
